Protocol architecture

The Dikenocracy framework is built through layered protocols. Each protocol defines a bounded part of the system and is designed to be version-aware, testable, and auditable. No higher-layer protocol may violate a lower-layer constraint.

Protocols are identified by their layer prefix (DKP-0, DKP-1, …) and a functional code. Each document carries a version number and a status flag (Draft, Candidate, Architecture Lock, Freeze).

L0 — Physical Truth

The foundation layer. Defines how observable physical reality is captured, verified, and made available to higher layers. No governance action may depend on data that bypasses this layer.

DKP-0-ORACLE-001 · v1.1 · Architecture Lock Candidate
Physical Truth Layer Protocol (PTL)

Defines the sole admissible interface between physical reality and the governance system. Covers oracle admissibility, confidence weighting, divergence detection, and systemic halt conditions.

DKP-0-TIME-001 · v1.0
Dikenocratic Time & Date Index Protocol (DTI)

Establishes a non-religious, event-neutral, deterministic day-counting system. Canonical unit: DTI-Day = JDN. Governance year = 360 days. Conversion rules to and from Gregorian are normative and machine-verifiable.

L1 — Core

Defines foundational axioms, identity attribution, scope boundaries, and system invariants that all higher layers must respect.

DKP-1-AXIOMS-001 · v1.0
Axioms Protocol

Non-negotiable invariant constraints that bound all measurement, optimization, and enforcement logic. No calculated benefit overrides an axiom.

DKP-1-IDENTITY-001 · placeholder
Subject Identity Protocol

Defines the legal and causal attribution of responsibility to originating Subjects. Full specification to be published.

L2 — Economic

Covers dual-circuit economy rules, tokenized value chains, conditional emission, and stake-based security mechanisms.
